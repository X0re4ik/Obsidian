## Про макрос error
Директива `#error` позволяет нам остановить процесс препроцессинга и сообщить об ошибки компиляции, чаще всего данная директива используется в связке с `#if`, например:
>main.cpp
```C++
// main.cpp

#if defined(__cplusplus)
    #error "This code requires a C++ compiler!"
#endif

int main() {
  return 0;
}
```
>terminal (`g++ -E main.cpp`)
```bash
# 0 "main.cpp"
# 0 "<built-in>"
# 0 "<command-line>"
# 1 "/usr/include/stdc-predef.h" 1 3 4
# 0 "<command-line>" 2
# 1 "main.cpp"
main.cpp:4:6: error: #error "This code requires a C++ compiler!"
    4 |     #error "This code requires a C++ compiler!"
      | 
```
## Про макрос pragma
Директива `#pragma` позволяет передать компилятору различные нестандартные, специфичные для конкретного компилятора функции. Особенность данной директивы в том, что, если компилятор встречает неизвестную ему функцию/команду, то он просто ее игнорирует. Так, например, `??` будет проигнорирована компилятором g++, но для компилятора clang выполниться оптимизация.
Ключевой и самой популярной командой является `#pragma once` - данная операция гарантирует, что заголовочный файл будет включен в пепрцессинг **единожды**. `#pragma once` заменяет громоздкий [[Include Guards]]
