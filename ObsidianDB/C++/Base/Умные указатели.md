## Почему мы не любим обычные указатели
1) Указатели ничего не говорят об объекте, на который указывают, то есть указатель на массив идентичен указателю на любой другой объект, а удаление происходит по разному
2) Отсутствие обработки ошибки при выбрасиывании исключения
```C++
int *x = new int{5};
foo(x); // throw exception
delete x;
```
3) Нет встроенного механизма выявления висячих указателей, так как delete "освобождает" память, то есть отдает владение ОС, но адрес, на который ссылается указатель все еще остается валидным
```C++
// Плохо
int *x = new int{5};
delete x;
if (x != nullptr) {/* Что-то делаем */}


// Хорошо
int *x = new int{5};
delete x;
x = nullptr; // Помечаем x, как указатель в никуда
```

## Какие виды владения существуют и какие реализованы внутри C++
1) Семантика значения (уникальное владение)
2) Запрет копирования и перемещения (уникальное владение) (**Scope Pointer**)
3) Запрет копирования (Уникальное владение) (**Unique Pointer**)
4) Подсчёт ссылок в контрольном блоке (Совместное владение) (**Shared Pointer**)
5) Интрузивный подсчёт ссылок (Совместное владение) (**Inclusive Pointer**)

Стандартная библиотека C++ предоставляет следующий набор умных указателей:
- std::auto_ptr
- std::unique_ptr
- std::shared_ptr
- std::weak_ptr
std::auto_ptr это наследство от std::unqiue_ptr, оставшееся от C++98, когда не было явления семантики перемещения

## О виртальном деструкторе
При динамическом полиморфизме деструктор обязан быть виртуальным, иначе наследники не смогу освободить свои ресурсы.
```C++
class A {
public:
	virtual ~A() = default;
};

class B : public A {
public:
	~B() { std::cout << "~B" << std::endl; }
};

class C : public A {
public:
	~C() { std::cout << "~C" << std::endl; }
};

int main() {
	std::vector<std::unique_ptr<A>> _vec;
	_vec.push_back(std::make_unique<B>());
	_vec.push_back(std::make_unique<C>());
	return 0;
}
```
Причем, в C++ деструктор это единственный виртуальный метод, который **обязан иметь реализацию**, иначе ошибка компиляции (**undefined reference**)
```C++
// В C++11 для интерфейса можно написать, что позволит компилятору реализовать дефолтный контруктор
virtual ~A() = default;
```
