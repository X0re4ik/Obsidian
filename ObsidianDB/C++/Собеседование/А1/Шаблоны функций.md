**Что такое шаблон функции? Зачем он нужен? Напиши пример простой шаблонной функции.**

Шаблон функции - это правило для генерации функции.
Компилятор генерирует код функции при первом вызове шаблонной функции, данный процесс именуется *инстанцирование*.
Шаблон функции позволяет сконцентрироваться на алгоритме, который может быть применен для различных типов данных.
Реализация шаблонных функций должна быть строго в hpp файле (исключеним является техника `explicit template instantiation`), так как компилятору необходимо знать сможет ли он вообще произвести *инстанцирование*

Пример простой функции шаблонной функции:
```C++
template<typename T, typename U>
auto sum(T a, U b)
{
	return a + b;
}
```



**Что такое инстанцирование шаблонной функции?**

Инстанцирование шаблонной функции - это процесс генерации кода под различные типы данных функции.
Инстанцирование бывает двух видов:
* `implicit` - неявное
* `explicit` - явное
Пример неявного (`implicit`) инстанцирования:
```C++
template<typename T, typename U>
auto sum(T a, U b)
{
	return a + b;
}

sum(1, 2); // Инстанцирование в функцию sum(int, int)
sum(1.0, 2); // Инстанцирование в функцию sum(double, int)
```
Пример явного (`explicit`) инстанцирования:
```C++
// .hpp
template<typename T, typename U>
auto sum(T a, U b);



// .cpp

template<typename T, typename U>
auto sum(T a, U b)
{
	return a + b;
}

template auto sum<std::string, std::string>(std::string a, std::string b);

sum(std::string("s"), std::string("s")); // Инстанцирование в функцию sum(std::string, std::string)
sum(1, 2); // Ошибка линковки, так как нет ссылки на функцию sum(int, int)
```


**Назови недостатки использования шаблонных функций.**

Из недостатков можно перечислить:

1) Увеличение времени компиляции
Процесс инстанцирования, так же невозможность вынести в бибиотечную реализацию часть кода (в `.cpp` файл), приводит к тому, что компилятору снова и снова приходится выполнять долгие операции с шаблонами.

2) Раздувание бинарного файла
Вызов функции с новыми типами приводит к генерации новой функции с новыми параметрами, что увеличивает количество кода и, тем самым, увеличивает размер финального бинарного файла

3) Сложность дебагинга ошибок инстанцирования
Если тип не поддерживает операцию `+`, то сообщение об ошибке будет довольно не очевидным, так как побольше части содержит код инстанцирования, которого по факту в программе и нет.

4) Невозможность скрыть реализацию от внешнего пользователя
