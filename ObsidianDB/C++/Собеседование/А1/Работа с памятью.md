
**Чем отличается `malloc` от `new`?**

| Параметр                   | malloc (`C` функция)                                                                                        | new (`C++` оператор)                                                                                                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Возвращает                 | `void*` - указатель на первый байт аллоцированной кучи. Требует явного приведения типов.                    | `A*` - указатель на объект. Конструктор уже будет вызван.                                                                                                                             |
| Выделение памяти           | Необходимо самому указать сколько памяти выделить под объект, так как `malloc` ничего не знает типе объекта | Знает о типе объекта и выделяет память под него                                                                                                                                       |
| Ошибка выделения памяти    | Возвращает `NULL`                                                                                           | Выбрасывает исключение `std::bad_alloc`. Данное поведение можно изменить при помощи `std::nothrow`( `new (std::nothrow) int[92233720368547758]`) - это позволит вернуть `nullptr`<br> |
| Освобождение               | `free` без вызова деструктора                                                                               | `delete` с вызовом деструктора                                                                                                                                                        |
| Возможность переопредления | Нет, malloc это функция из мира `C`. Возможно только написать свой аллокатор                                | Да, так как это оператор, а не функция, то можно переопредлить оператор `new` как для одного конкретного класса, так и для всей программы в целом.                                    |
| Зачем нужен                | Только выделения блока памяти под объект                                                                    | Выделения блока памяти под объект и его создание через конструктор объекта                                                                                                            |
Отметим, что нельзя смешивать `free` с `new` и `delete` с `malloc`, так как
1) `new` и `free` могут использовать разные менеджеры памяти
2) У объекта не вызовется деструктор, что может привести к учетки и `UB`
Еще одна фишка `new` - `placement new` - возможность выделения памяти в опредленном месте.

Например:

```C++
void* memory = operator new(sizeof(MyClass));
MyClass* obj = new(memory) MyClass(100, "Пример");
obj->~MyClass();
operator delete(memory);
```


**Как `new` понимает, сколько памяти надо выделять?**

Данный вопрос можно разбить на 2 части:

1) Создание протсого объекта
При создании объекта, компилятор заранее знает сколько будет весить тот или иной объект и, соответсвенно, может сказать сколько памяти выделить.  Но есть нюанс с выравниванием, так как память проще читать по блокам кратный размеру типа.
Таким образом new генерирует вызов `operator new(sizeof(A))`.

Пример:
```C++
struct A {
	int x; // 4 байт
	int y; // 4 байт
};

A* a = new A; //  Компилятор знает, что A = 4 + 4 = 8 байт -> (operator new(8))
```

2) Создание массива
Здесь все чуть интереснее, при выделении памяти под массив выделяется чуть больше памяти (overhead) - под размер самого массива, стандарт не гарантирует где и как будет описан размер массива, так это является деталью реализации.

Пример:
```text
Адрес в памяти (возрастает ->)
[Служебный заголовок] [ arr[0] ] [ arr[1] ] [ arr[2] ]
^                     ^
|                     |
0x1000                0x1010  <- Указатель, который вы получили
(хранится N=3)        (возвращается вам)
```

**Откуда `delete[]` знает, сколько элементов надо удалят**

При выполнении `new[]` в на самом деле выделяется чуть больше байт, чем размер массива, эти байты содержат дополнительную информацию о выделенном массиве в т.ч. и размер.

Оператор `operator delete[]`:
1) Читает информацию из служебного заголовка
2) Вызывает деструктор для N объектов
3) Разом очищает блок памяти размером заданного в служебной информации. Здесь стоит пояснить, что мы не можем очистить `N * sizeof(A)`, так как существует выравнивание
