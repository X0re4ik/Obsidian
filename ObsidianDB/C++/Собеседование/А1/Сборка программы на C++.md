**Расскажи кратко про каждый этап компиляции программы на С++**

Этап компиляции программы можно разделить на 4 этапа:
1) Препроцессинг
2) Компиляция
3) Ассамблирование
4) Компоновка

Рассмотрим каждый из этапов чуть более подробно:

* Препроцессинг

На данном этапе мы раскрываем все макросы и включаем все `include` директивы, на выходе получаем один большой файл, содержащий написанный нами код.
Отметим, что на данном этапе мы не проверяем синтаксис и семантику, только готовим код к дальнейшему построению `AST`-дерева.
```bash
>> g++ -E main.cpp -o main.ii
```

* Компиляция

Компиляция - этап состоящий из множества довольно сложных операция, перечислим несколько из них
1) Построение `AST`-дерева (Frontend) 
2) Проверка синтаксиса (Frontend)
3) Проверка семантики (Frontend)
4) Формирование ассемблерного файла (`Middle-end` + `Backend`)
Отметим пункт 4 состоит из предварительного создания `RL` - кода и множества применений оптимизации.
```bash
>> g++ -S main.ii -o main.s
```

* Ассемблирование

До этого этапа мы все еще работали с абстракциями, но здесь мы уже имеет ассемблерный файл под нашу конкретную архитектуру. Теперь на основании ассемблерного файла мы можем сформировать объектный файл.
```bash
>> g++ -c main.s -o main.o
```

* Компоновка

Перед линкером находится несколько объектных файлов, некоторые из них могу являться статическими библиотеками, некоторые динамическими. Каждый объектный файл содержит таблицу символов - таблица, в которой описаны какие переменные и методы скрывает объектный файл. Линкер проходится по всем объектам и формирует единый файл, содержащий все вызовы необходимых инструкций. 
```bash
>> g++ main.o -o main
```

**Что такое define guard? Какие виды define guard'ов ты знаешь?**

`define guard` - это механизмы препроцессора, которые защищают от многократного включения одного и того же заголовочного файла в единицу трансляции
Перечислим самые популярные:
1) `Include Guard`
Реализуется через команды `ifndef`, `define`, `endif`:
```C++
# ifndef MY_HEADER_HPP
# define MY_HEADER_HPP 1
struct MyHeader {};
# endif MY_HEADER_HPP
```
2) `pragma once`
Официально поддержки в языке так и нет, но самые популярные компиляторы `GCC`, `Clang` и `MSVC` поддерживают данную директиву.


**Что такое макрос? Напиши пример макроса, который вычисляет максимальное из двух чисел.**

Макросы - это конструкция языка, которая отрабаотывает только на этапе препроцессинга и далее в жизни `C++` приложения не участвуют. Необходимы, в первую очередь, для условной компиляции (`ifndef`), кодогенерации и добавление отладочной информации (`__FILE__`, `__LINE__`)
```С++
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

**Какая разница между статической и динамической библиотеками?**

Статическая и динамическая библиотека - это 2 совершенно разные библиотеки, которые собираются и линкуются по-разному.

**Статическая библиотека** - это архив объектных файлов, который физически встраивается в финальный исполняемый файл на этапе компиляции, соответственно, статическая библиотека очень сильно влияет на финальный размер бинарного файла, но исключает накладные ресурсы `RAM` при своей работе.

**Динамическая библиотека** - библиотека, которая подгружается загрузчиком на момент старта приложения (или вызывается напрямую из программы). Динамические библиотеки так же формируются из объектных файлов, но отличительная их черта в том, что компилятор не знает адреса функций и переменных этой библиотеки заранее, поэтому вынужден формировать таблицу смещений и таблицу символов. Не влияют на размер бинарного файла, но съедают `RAM`, так как загружаются в память. Стоит отметить, что сама билиотека загружается в память в одном экземпляре, то есть 10 приложений будут использовать 1 инстанс этой библиотеки.
